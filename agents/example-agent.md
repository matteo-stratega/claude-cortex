## How to Use This Template

1. **One agent = one domain.** If two agents could answer the same question, merge them.
2. **Be specific about personality.** "Professional" means nothing. "Direct, no filler, uses bullet points, never says 'great question'" means something.
3. **Define triggers.** When should this agent load? Technical tasks? Sales calls? Content work?
4. **Test the boundary.** Ask: "Does this agent need its own memory and personality that would conflict with another agent?" If no, it's a protocol inside an existing agent, not a new agent.
