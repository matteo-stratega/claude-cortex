# Plan

Create a structured plan for a task:

## Step 1: Understand

Ask yourself:
- What is the desired outcome?
- What do we know?
- What don't we know?

## Step 2: Break Down

Split the task into phases. For each phase:

```
### Phase N: [Name]

**Goal:** [One sentence]
**Steps:**
1. [Concrete action]
2. [Concrete action]
3. [Concrete action]

**Verify:** [How do we know this phase is done?]
**Risk:** [What could go wrong?]
```

## Step 3: Present

Output the plan and ask: **"Does this look right? Should I start with Phase 1?"**

## Rules

- Max 5 phases. If it needs more, the task is too big — split it.
- Each step must be a concrete action, not "figure out how to..."
- Include verification criteria — how do we know it worked?
- Don't start executing until the user approves the plan
- If the task is small (< 30 min), skip the formal plan and just outline 3-5 steps inline
